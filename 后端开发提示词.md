## 提示词内容

```
你是一名 Java/Spring Boot 后端开发工程师。请根据本项目根目录下的 `API接口文档.md`，使用 Spring Boot 3.x 开发一个后端项目，完整实现文档中定义的所有 RESTful API 接口。

## 技术栈要求

- **框架**: Spring Boot 3.2.x + JDK 17
- **构建工具**: Maven
- **ORM**: MyBatis (使用 mybatis-spring-boot-starter)
- **缓存**: Spring Data Redis (用于 Token 存储和热点数据缓存)

- **文件上传**: 阿里云 OSS (aliyun-sdk-oss)
- **API 文档**: SpringDoc OpenAPI (springdoc-openapi-starter-webmvc-ui)
- **工具库**: Lombok (减少样板代码)

## 数据库设计要求

请设计以下数据表：

1. **user** - 用户表 (id, username, password, avatar, bio, created_at, updated_at)
2. **post** - 帖子表 (id, title, description, content, image_url, type, author_id, likes_count, comments_count, created_at, updated_at)
3. **comment** - 评论表 (id, post_id, user_id, content, created_at)
4. **like** - 点赞表 (id, post_id, user_id, created_at) - 联合唯一索引(post_id, user_id)

请提供完整的 SQL 建表脚本放在 `src/main/resources/sql/schema.sql`。

## Redis 使用策略

- **JWT Token**: 将 access token 存入 Redis，设置 2 小时过期；refresh token 设置 30 天过期
- **Token 黑名单**: 用户登出时将 token 加入黑名单，过期时间同 token 剩余有效期
- **点赞计数**: 使用 Redis 缓存帖子点赞数，定时或按需同步到 MySQL
- **帖子列表缓存**: 热门帖子列表缓存 5 分钟

## JWT Token 设计

- **AccessToken**: 有效期 2 小时，包含 userId
- **RefreshToken**: 有效期 30 天，包含 userId
- 两个 Token 都存储在 Redis 中以便支持主动登出和强制失效

## 开发步骤

请按以下顺序逐步开发：

1. **初始化项目**: 创建 `pom.xml`、`application.yml`、启动类
2. **数据库**: 编写 `schema.sql` 建表脚本
3. **基础设施**: 配置 Redis、MyBatis、JWT、OSS
4. **基础类**: ApiResponse、BusinessException、GlobalExceptionHandler、JwtUtil
5. **用户模块**: Entity → Mapper → Service → Controller（注册/登录/刷新/资料/登出）
6. **帖子模块**: Entity → Mapper → Service → Controller（CRUD + 列表 + 我的帖子）
7. **评论模块**: Entity → Mapper → Service → Controller
8. **点赞模块**: Entity → Mapper → Service → Controller（含 Redis 缓存）
9. **上传模块**: Service → Controller（阿里云 OSS 上传）
10. **验证**: 使用 curl 或 Postman 测试所有接口

## 关键实现细节

### JWT 认证拦截器
- 从 `Authorization: Bearer <token>` 提取 token
- 验证 JWT 签名和有效期
- 检查 Redis 中 token 是否存在（已被登出）
- 验证通过后将 userId 存入 request attribute

### 密码安全
- 使用 BCryptPasswordEncoder 加密存储密码
- 登录时使用 matches() 验证密码

### 帖子列表分页
- 使用 MyBatis 的 `LIMIT #{offset}, #{size}` 实现分页
- 支持按 type 筛选、按 createdAt/likesCount 排序
- 返回 totalElements 用于前端分页计算

### 点赞功能
- 每次点赞/取消先更新 Redis 计数器
- 使用 Redis Set 记录用户点赞状态，防止重复点赞
- 帖子详情查询时从 Redis 读取最新点赞数
- 定时任务或请求触发时将 Redis 点赞数据同步到 MySQL

### 文件上传
- 接收 MultipartFile
- 生成唯一文件名（UUID + 时间戳 + 后缀）
- 上传到阿里云 OSS，返回可访问 URL
- 限制文件大小 10MB，仅允许图片格式

## 注意事项

1. 所有接口路径前缀为 `/api`，在 Controller 或 WebConfig 中统一配置
2. 不需要认证的接口：注册、登录、刷新 Token、帖子列表、帖子详情
3. 删除操作必须校验操作者身份（帖子仅作者可删，评论仅评论者可删）
4. 发帖时 description 若为空则自动从 content 截取前 50 字
5. 分页 page 从 0 开始，size 最大 50
6. 所有时间字段返回格式为 `yyyy-MM-dd HH:mm:ss`
7. 阿里云 OSS 配置项（accessKeyId、accessKeySecret、bucketName、endpoint）在 application.yml 中配置

## 使用方法

1. 在 Claude Code 中打开后端项目目录
2. 将上方「提示词内容」之间的内容完整复制并发送

## 额外配置

如果需要在项目中添加阿里云 OSS 配置，在 `application.yml` 中补充：

```yaml
aliyun:
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    access-key-id: YOUR_ACCESS_KEY_ID
    access-key-secret: YOUR_ACCESS_KEY_SECRET
    bucket-name: hanculture-bucket
    url-prefix: https://hanculture-bucket.oss-cn-hangzhou.aliyuncs.com/
```
